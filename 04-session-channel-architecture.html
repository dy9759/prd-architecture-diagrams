Session / Channel 重构架构

Session 表废弃 · Channel/Thread 统一 · ThreadContextItem 结构化 · 6 阶段迁移

核心模型迁移

  • • Thread.id 独立 UUID，不再依赖 root message ID
  • • 新增 workspace_id/root_message_id/issue_id/status
  • • ThreadContextItem 替代 session.context JSONB
  • • Message 从三路多态简化为 channel_id + thread_id
  • • reply_count 不再计入系统消息

废弃范围边界

  • • 本 PRD 仅涉及"产品协作会话"的 session 表
  • • 不涉及 remote_session / impersonation_session
  • • agent_task_queue.session_id 保留（Provider 会话）
  • • 每 legacy session → 1 private Channel + 1 Thread
  • • session_migration_map 支持兼容期写入转写

MediationService 统一路由

  • • 消除 message handler + mediation 双重触发
  • • auto_reply_config 仅作输入，由 MediationService 决策
  • • reply_slot：T+300/600/900 秒 SLA 升级链
  • • scope='conversation' System Agent 自动入所有 Channel
  • • 防循环：agent→agent 3 轮、50 次/24h 频率限制
