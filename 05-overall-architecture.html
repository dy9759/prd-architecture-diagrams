四 PRD 统一视图 · 6 层分层 · 跨模块绑定 · Channel / Agent / Project 三角核心

四 PRD 的整合主轴

  • • 核心三角: Project ↔ Channel ↔ Agent (通过 channel_id / runtime_id)
  • • Plan ↔ Thread: 生成计划的讨论线程绑定
  • • Slot ↔ Agent: 协作槽位绑定执行者，联动风险分级
  • • 所有状态变更 → events.Bus + activity_log (永久)
  • • 需人工响应 → inbox_item (+ severity/resolution)

双路径执行统一接口

  • • 共享 execution 队列 (FOR UPDATE SKIP LOCKED)
  • • 本地 Daemon: HTTP 轮询 + CLI Provider
  • • 云端 Executor: SDK Session + Sandbox + Lease
  • • context_ref 按 mode 记录物理事实
  • • MCP 工具 17 个 · 每次调用后端权限校验

RBAC 三维一体

  • • 管理权限: handler 层 requireAgentOwner 等守卫
  • • 功能权限: System=admin 级 · Personal 继承 User
  • • 风险分级: low/medium 直行 · high 需 slot · critical 需前置审批
  • • 数据权限: 后端按 workspace+user+agent+对象校验
  • • Impersonation 仅 Personal Agent · 30min · 消息双字段审计

数据生命周期统一策略

  • • retention_class: permanent / ttl / temp 三档
  • • approved Artifact = permanent · Review = permanent
  • • Execution 日志默认 90d TTL
  • • ThreadContextItem 按 type 默认策略
  • • activity_log 始终 permanent · 审计必需
