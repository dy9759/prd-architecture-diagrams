Cross-cutting 跨模块架构

事件 · 审计 · 服务契约 · MCP 工具 · Inbox · 密钥 · 配额 · 错误码 · 可观测性

三大新服务

  • • PlanGeneratorService：Thread → LLM → Plan/Task/Slot
  • • MediationService：消息路由 + SLA 升级链 + 防循环
  • • CloudExecutorService：SDK Session + Sandbox 管理 + Lease
  • • 所有服务通过 events.Bus 协作
  • • 每个关键事件必须写 activity_log (同事务)

事件 vs 审计 分工

  • • WebSocket = best-effort 实时推送 (非持久)
  • • activity_log = 保证一致的永久审计 (真相源)
  • • 断线重连后前端主动拉取，不依赖事件补发
  • • inbox_item = 需要人工响应的通知入口
  • • reply_slot = Thread 级 SLA 跟踪 (300/600/900 秒)

基础设施 MVP

  • • 17 个 MCP 工具后端 handler + 每调用权限校验
  • • workspace_secret：AES-256-GCM 主密钥环境变量
  • • workspace_quota：execution claim 前检查 USD 上限
  • • 错误码分类：DOMAIN_NAME + retriable 标记
  • • Metrics：Prometheus 风格 12 个核心指标
